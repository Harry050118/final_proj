# OpenClaw Related Work 深度评测项目入口

> 这份文档是小组阅读入口，只保留项目概览、导航、分工和下一步。详细协议已按职责拆到 `docs/` 目录。

## 项目一句话

我们选择期末 Project 的 **任务 B：智能体框架缺陷诊断（Research 赛道）**，评测 OpenClaw 在开放学术检索条件下生成 Related Work 的可靠性。

核心问题：

> 在去掉论文 title、Related Work、references、figures、tables 和 captions 后，OpenClaw 能否根据剩余正文识别研究方向、检索相关文献，并生成低幻觉、可复查、结构合理的 Related Work？当检索结果混入无关或误导论文时，它是否会误引？

## 总体路线

1. 选择 3-5 篇 pilot 论文。
2. 构造匿名化输入：保留正文核心信息，删除 title、Related Work、references 和图表线索。
3. 运行三类输入条件：Clean Search、Noisy Search、Misleading Search。
4. 对比三组方法：No Intervention、Generic Writing Prompt、Citation-Grounding Workflow。
5. 按 citation reliability、检索质量、抗干扰能力和写作质量进行人工评分。
6. 汇总典型成功与失败案例，完成前端展示、最终报告和 PPT。

## 协作原则

- 所有实验必须保存输入、prompt、输出、query 和候选论文。
- 每次改 prompt 或 workflow 都要记录版本。
- 标注分歧不能直接平均，需要记录原因。
- 最终报告必须展示成功案例和失败案例。
