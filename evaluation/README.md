# RWEval Integration

This directory contains the local `Spiritswin/RWEval` pipeline adapted for this project.

## Layout

- `RWEval/`: upstream evaluation pipeline, adapted to use DeepXiv instead of Semantic Scholar.
- `RWEval/examples/`: upstream sample inputs for trying the original pipeline shape.
- `../runs/clean_search/sampleN/`: project sample directories that follow the RWEval native four-file format.

## Environment

Copy `RWEval/.env.example` to `RWEval/.env` and fill the API settings before running evaluation:

```text
LLM_API_BASE_URL=
LLM_API_KEY=
LLM_MODEL=
DEEPXIV_TOKEN=
RW_EVAL_CACHE_DIR=.cache/rw_eval
```

RWEval expects an OpenAI-compatible `POST /chat/completions` endpoint. Citation normalization and claim evidence retrieval use DeepXiv by default. If the LLM or DeepXiv token is missing, use `--no-deepxiv` only for smoke tests; do not treat that as a complete evaluation.

## Project Sample Format

Each project sample should live under `runs/clean_search/sampleN/` and match the native RWEval file layout:

```text
runs/clean_search/sample1/
  s_text.txt
  s_reference.txt
  g_text.txt
  g_reference.txt
  trace.md        # optional audit file; RWEval ignores it
```

`s_text.txt` and `s_reference.txt` are generated by `/openclaw-related-work-eval`. `g_text.txt` and `g_reference.txt` are supplied after the OpenClaw run and must not be read by the skill. `trace.md` records DeepXiv queries and citation checks for reproducibility, but it is not an RWEval input file.

## Evaluate A Project Sample

From the project root:

```powershell
cd evaluation\RWEval
python -m rw_eval.cli evaluate-files `
  --directory ..\..\runs\clean_search\sample1 `
  --sample-id sample1 `
  --output ..\..\runs\clean_search\sample1\report.json `
  --markdown-output ..\..\runs\clean_search\sample1\report.md
```

The command reads the four txt files, writes `runs\clean_search\sample1\sample_input.json`, then writes the report files:

```text
runs\clean_search\sample1\sample_input.json
runs\clean_search\sample1\report.json
runs\clean_search\sample1\report.md
```

Repeat the same command with `sample2`, `sample3`, and so on for later runs.

## Run Upstream Examples

From the project root:

```powershell
cd evaluation\RWEval
python -m rw_eval.cli evaluate-files `
  --directory examples\sample1 `
  --output outputs_local\sample1_report.json `
  --markdown-output outputs_local\sample1_report.md
```

Repeat with `examples\sample2` to test the second upstream sample.
