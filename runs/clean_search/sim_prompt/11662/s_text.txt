\section{Related Work}

\subsection{Verifiers and Reward Models for AI Agents}

Verifiers---functions that assign scalar or textual rewards to candidate behaviors---have long played a central role in AI progress. In classical game-playing settings, verifiers based on game outcomes (win/loss signals) guided both search and learning in Go~\cite{silver2016mastering} and Chess~\cite{silver2018general}. More recently, formal verifiers such as unit tests and exact-match checks have driven breakthroughs in code generation and mathematical reasoning through reinforcement learning with verifiable rewards (RLVR)~\cite{wang2024deepseek, guo2025deepseek}. However, these successes rely on domains with well-defined success criteria, and extending verifier-driven progress to open-ended settings---where tasks admit multiple valid solutions and evaluation requires nuanced, multimodal judgment---remains an open challenge.

To bridge this gap, a growing body of work has explored using large language models and multimodal large language models (MLLMs) as general-purpose evaluators of agent behavior. Agent-RewardBench~\cite{chen2025agentrewardbench} introduced a step-level, multi-dimensional benchmark for assessing MLLMs as reward models across perception, planning, and safety in real-world agent scenarios, revealing that even top-performing models achieve only modest accuracy. Agentic Reward Modeling~\cite{cao2026agentic} proposed shifting from passive LLM-as-a-Judge evaluation to proactive environment interaction, equipping verifier agents with tools to probe latent system states. GUI-Shepherd~\cite{wang2025guishepherd} introduced a process reward model trained on diverse GUI interaction data to provide dense, step-by-step feedback for long-sequence GUI tasks. DeltaRubric~\cite{jiang2025deltarubric} proposed rubric-based multimodal reward modeling to mitigate lazy judging in single-step evaluators. Complementing these efforts, MagicGUI-RMS~\cite{wang2026magicguirms} employs a multi-agent reward model system for self-evolving GUI agents. Collectively, this line of work underscores both the promise and the difficulty of using MLLMs as reliable verifiers for agent behavior in open-ended domains.

\subsection{Agent Benchmarks and Environments}

The study of autonomous agents has been enabled by a series of increasingly realistic benchmarks. WebArena~\cite{zhou2024webarena} introduced a self-hosted web environment spanning e-commerce, social forums, and development domains with 812 long-horizon tasks evaluated via functional correctness. VisualWebArena~\cite{koh2024visualwebarena} extended this paradigm to multimodal settings with 910 visually grounded web tasks requiring combined text and image understanding. OSWorld~\cite{xie2024osworld} provided the first scalable, real-computer-environment benchmark for multimodal agents across Ubuntu, Windows, and macOS, featuring 369 tasks involving multi-app workflows. For robotics, robomimic~\cite{mandlekar2021robomimic} established a framework and benchmark for imitation learning in robot manipulation, including the challenging tool-hang task studied in this work. The BrowserGym ecosystem~\cite{drouin2024browsergym} addressed the need for unified evaluation infrastructure for web agents, while Windows Agent Arena~\cite{bonatti2024windows} evaluated multi-modal OS agents at scale. These benchmarks collectively provide the diverse environments---web, desktop, and robotic---necessary for a comprehensive study of MLLM-based verification.

\subsection{LLM-as-Judge and Evaluation Biases}

Using LLMs as evaluators (LLM-as-a-Judge) has become a widely adopted paradigm for scalable assessment, but several systematic biases have been identified. Position bias---the tendency for LLM judges to favor responses based on their order in the prompt---was documented by~\cite{wang2023largellmjudge}, who proposed calibration frameworks involving evidence-based scoring and balanced position calibration. Self-bias, where models favor their own generated outputs during self-evaluation, was formalized by~\cite{xu2024pride} and shown to amplify during self-refinement pipelines. Relatedly,~\cite{pan2024selfpreference} introduced a quantitative metric for self-preference bias rooted in algorithmic fairness, demonstrating that LLMs assign higher ratings to texts with lower perplexity---suggesting that familiarity with self-generated content drives the bias. The broader reliability of LLM judges has been examined by~\cite{zheng2024humansllmjudge}, who studied judgment biases comparing human and LLM evaluators. Our work identifies a distinct form of bias---agreement bias---that differs from these previously documented biases: it manifests as a systematic tendency to over-validate \textit{agent behavior} in multimodal settings, persists despite the verifier receiving externally grounded information (e.g., screenshots with Set-of-Marks), and occurs even when the verifier and agent are built from entirely different model families.

\subsection{Self-Improvement and Iterative Refinement for Agents}

A growing body of work has explored enabling language agents to improve through iterative self-correction. Reflexion~\cite{shinn2023reflexion} introduced a verbal reinforcement learning framework where agents generate reflective textual summaries from environmental feedback and store them in episodic memory to guide future decisions, achieving significant gains without model fine-tuning. Self-Rewarding Language Models~\cite{yuan2024selfrewarding} proposed an iterative alignment framework where a language model simultaneously improves instruction-following and self-generated reward modeling by acting as its own judge. For web agents specifically,~\cite{patel2024llmsselfimprove} demonstrated that LLMs can self-improve on WebArena tasks through fine-tuning on synthetic trajectories filtered by self-critique and environmental signals. Agent-R~\cite{wang2025agentr} trained language model agents to reflect via iterative self-training, while STeCa~\cite{han2025steca} proposed step-level trajectory calibration for LLM agent learning. These methods share a common dependency: they rely on a verifier (whether heuristic, environmental, or model-based) to judge trajectory quality and provide feedback. Our work examines the risks when this verifier is itself an MLLM prone to agreement bias, and shows that improving verification quality yields compounding benefits in self-improvement pipelines.

\subsection{Test-Time Scaling, Reasoning, and Process Supervision}

Several techniques have been developed to improve LLM reasoning at inference time. Chain-of-thought (CoT) prompting~\cite{wei2022chainofthought} elicits intermediate reasoning steps to improve complex reasoning performance. Self-consistency~\cite{wang2022selfconsistency} samples multiple reasoning paths and aggregates via majority voting, effectively using the mode of the model's output distribution as a verifier. Process reward models (PRMs) provide fine-grained feedback on intermediate reasoning steps rather than only final outcomes. AgentPRM~\cite{banerjee2025processreward} introduced a lightweight framework for training LLM agent PRMs using Monte Carlo rollouts, while GenPRM~\cite{zhang2025genprm} scaled test-time compute for PRMs via generative reasoning. Verifiable Process Reward Models~\cite{li2025verifiableprocess} explored process supervision with verifiable correctness signals. Our findings reveal that these test-time scaling techniques---CoT, majority voting, and reasoning models---are insufficient to resolve agreement bias, because the bias is rooted in the model's output distribution itself: when TNR is low, the probability of sampling a correct verification on failure subsets is at or below chance, rendering mode-seeking strategies ineffective.

\subsection{Visual Grounding and GUI Agents}

Effective verification of digital agent behavior requires robust visual understanding of graphical user interfaces. Set-of-Mark (SoM) prompting~\cite{yang2023setofmark} introduced a visual prompting method that overlays images with interpretable region marks, significantly enhancing the visual grounding capabilities of large multimodal models. SeeClick~\cite{cheng2024seeclick} advanced GUI grounding for visual GUI agents through pretraining on large-scale screenshot data. UI-TARS~\cite{qin2025uitars} pioneered native GUI agent models that operate solely on screenshots, unifying perception, reasoning, and action in an end-to-end architecture and achieving state-of-the-art performance across GUI benchmarks. WebVoyager~\cite{he2024webvoyager} built an end-to-end web agent with large multimodal models. These advances in visual grounding and GUI agent design provide the foundation upon which MLLM-based verifiers must operate, as they must process and reason over multimodal trajectory data---screenshots, DOM trees, and action sequences---to produce reliable evaluations.
